# Visual Species Shift Execution Plan v2
## Status
- **Document Type:** Execution plan
- **Authority Level:**
  - Subordinate to `Brand_Identity_Brief_v0.1`
  - Subordinate to `VI_System_Brief_v0.2_locked`
  - Subordinate to `Visual_Contract_Freeze_v0.1`
  - Guided by `Visual_Species_Shift_Brief_v0.1`
- **Purpose:**
  - Translate the species-shift brief into a practical implementation sequence
  - Replace current publication surface with an AI-era Protocol Architect brand surface
  - Prevent overcorrection into empty brutalism or generic future tech styling

## 0. Core Diagnosis
Phase 0.5 proved that structural logic and componentization alone are insufficient. The site can still feel like a polished knowledge site. This execution phase is a **species-shift implementation wave**.

## 1. Goal Description
Build a brand surface that can only plausibly belong to **Jearon Wong — Protocol Architect for the Agent Era**.
- person-first, thesis-first, proof-visible
- protocol-grade, founder-grade, systems-native

## 2. Non-Goals
This phase must NOT drift into brutalist styling, generic AI SaaS futurism, literary aesthetics, generic founder branding, or preserve card-led publication DNA.

## 3. High-Level Execution Sequence
### Phase 1 — Protocol Authority Typography
Rebuild typographic hierarchy to communicate systems authority. (Push strong, not brutalist).
### Phase 2 — Portrait as Architectural Anchor
Transform the portrait area into a true identity anchor, not a monolithic dead slab.
### Phase 3 — Card Species Conversion
Convert content cards from cozy publication cards into architectural panels / signal surfaces.
### Phase 4 — Homepage Rhythm Reconstruction
Create visible hierarchy shocks and non-flat section cadence.
### Phase 5 — Identity & Archetype Validation
Validate recognition and archetype rejection.

## 4. Phase 1 — Protocol Authority Typography
- **Primary Display:** Modern sans/grotesk. Sharp and deliberate.
- **Font Weight:** 800/900 allowed selectively for major declarations. Not every heading.
- **Tracking:** Negative tracking allowed selectively for high-level declarations to improve authority/compression.
- **Case:** Uppercase allowed for micro labels / eyebrows. NOT default for thesis.

## 5. Phase 2 — Portrait as Architectural Anchor
- Portrait must be visually large and structurally integrated.
- Fallback must use a locked identity treatment (Monogram, texture, architectural frame).
- Must NOT be a tiny side card or an empty slab with no identity.

## 6. Phase 3 — Card Species Conversion
- Eliminate cozy publication-card language.
- Convert to: proof slabs, signal panels, structured blocks, open-space grids with hard alignment.
- Allowed: Thin lines, sharp corners, stronger typographic structure.

## 7. Phase 4 — Homepage Rhythm Reconstruction
Break flat cadence and replace with: declaration -> signal -> drop -> pressure -> proof -> editorial continuation.
Each transition must visibly change scale, density, contrast, or width logic.

## 8. Phase 5 — Identity & Archetype Validation
Validate 3s Person Test, 5s Thesis Test, 10s Proof Test. Reject if misidentified as old blog, literary, generic founder, or SaaS archetype.

## 9. Open Decisions — Locked for v2
- **Typography:** stronger weight, sharper hierarchy, selective compression, **no full-brutalist takeover**.
- **Cards:** card species conversion, no total card elimination, keep structural scanability.
- **Portrait:** portrait as architectural anchor, no giant dead slab.
