# JearonWong.com VI System Brief v0.4 — Architectural Registry

### Status
- **Document Type:** Highest Authority / Strategic Pivot
- **Persona Alignment:** Protocol Architect for the Agent Era
- **Core Philosophy:** Evidence First, Identity as Infrastructure, Restrained Authority.

## 1. The Strategic Object
JearonWong.com is NOT a blog or a portfolio. It is an **Architectural Headquarters**.
It exists to establish a **Proof Chain** for the thesis: "How AI agents reliably deliver real work."

## 2. Visual Identity (VI)

### 2.1 Mood: The "Black Box" of Protocol
- **Objective/Precise**: The site should feel like an industrial control surface or a high-end technical registry.
- **Evidence-Led**: Data, metadata, and cross-references are more important than large decorative images.
- **Restrained**: No "self-crowning" animations. Motion is used only to show system state or hierarchy focus.

### 2.2 Color Palette: Monolithic & Signal
- **Base**: `Monolith Charcoal` (#111111) — Solid, grounded.
- **Surface**: `Anodized Slate` (#1a1a1a) — For nested registry modules.
- **Text**: `Machine White` (#f1f5f9) — High contrast for core declarations.
- **Signal Accents**: 
  - `Registry Blue` (#3b82f6) — For links and navigation.
  - `Evidence Cyan` (#22d3ee) — For data points and proof markers.
  - `Status Gray` (#64748b) — For metadata and annotations.

### 2.3 Typography: Technical & Tabular
- **Display**: `Outfit` (Bold, structural, used for declarations).
- **Body**: `Inter` (Precise, high readability).
- **Metadata**: `JetBrains Mono` (Always used for registry keys, timestamps, and proof signals).

## 3. UI/UX Hierarchy

### 3.1 Homepage: The Brand Manifesto
The homepage must satisfy the "Three Truths" in order:
1. **WHO**: The Protocol Architect (Headshot is a system anchor).
2. **WHAT**: The Defining Thesis (The core gap in agent work).
3. **PROOF**: The Evidence Ledger (The projects as concrete thinking).

### 3.2 Taxonomy Re-mapping
- **Identity Spec (About)**: A technical breakdown of Jearon Wong's role, background, and operational philosophy.
- **Evidence Ledger (Projects)**: A "Proof Room" where each project is an auditable record of the architect's thinking.
- **Registry / Logbook (Essays)**: A linear log of thoughts and system updates.
- **Framework Expansion (Ideas)**: Deep-dives into the core pillars of the protocol.

## 4. Forbidden Patterns
- No generic SaaS cards.
- No "Personal Brand" fluffy copy.
- No decorative gradients that serve no functional purpose.
- No "Portfolio" DNA (Projects are proof, not work-for-hire).
- No "Blog" DNA (Essays are registry entries, not just posts).
