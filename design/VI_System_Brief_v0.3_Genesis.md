# JearonWong.com VI System Brief v0.3 — Genesis Protocol

### Status
- **Document Type:** Canonical VI authority
- **Authority Level:** Superior to all page/UI implementation decisions
- **Applies To:** Full-site UI/UX convergence
- **Purpose:** Transition the brand from a rigid "Evidence Ledger" to a premium "Cognitive Blueprint".

## 1. Core Visual Direction
The **Genesis Protocol** system is designed to evoke the feeling of a high-end AI runtime. It moves away from flat, brutalist surfaces toward depth, transparency, and kinetic energy.

### 1.1 Brand Mood
- **Premium Tech**: Feeling like a high-end developer tool or a futuristic blueprint.
- **Cognitive Depth**: Using layers and transparency to suggest complex intelligence.
- **Kinetic Energy**: Using gradients and subtle motion to suggest active work delivery.

## 2. Visual Language

### 2.1 Color Palette (Genesis)
- **Primary Background**: `Deep Space Navy` (#020617)
- **Secondary Surface**: `Submerged Slate` (#0f172a)
- **Glass Tint**: `rgba(15, 23, 42, 0.6)` with 20px blur.
- **Kinetic Accent**: Gradient Mesh (`#22d3ee` -> `#8b5cf6`).
- **Text (High)**: `Frost White` (#f8fafc)
- **Text (Muted)**: `Silver Slate` (#94a3b8)

### 2.2 Geometry & Materials
- **Radius**: `12px` (Standard for all panels and buttons).
- **Borders**: `1px solid rgba(255, 255, 255, 0.1)` with 0.5px internal glow.
- **Materials**: Glassmorphism is the default for floating elements (Header, Signal Cards).

### 2.3 Typography
- **Display**: `Outfit` (Geometric, clean, wide tracking allowed for headlines).
- **Body**: `Inter` (Standard, high readability).
- **Mono**: `JetBrains Mono` (Technical, precise).

## 3. UI/UX Components

### 3.1 Genesis Stage (Hero)
The hero section is no longer a flat split. It is a "Stage" with:
- A mesh gradient background.
- Floating glassmorphic cards.
- Integrated grid lines that pulse subtly.

### 3.2 Cognitive Panels (Cards)
Cards for Ideas and Projects must:
- Use the 12px radius.
- Have a subtle outer glow on hover.
- Use internal gradients to guide the eye toward the "Proof" or "Thesis".

### 3.3 Floating Header
The site header is a floating glass bar with a blurred background, staying visible during scroll to provide constant identity context.

## 4. Implementation Rules
- **No pure black**: Use the `#020617` Navy base.
- **No 0px radius**: Use the `12px` Genesis radius.
- **No flat dividers**: Use soft gradients or spacing to separate content.
- **Preserve Authority**: The "Protocol Architect" role line remains the core declaration.
