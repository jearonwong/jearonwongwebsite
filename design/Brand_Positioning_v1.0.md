### The Argument

Prompt Engineering and Context Engineering solve the input side.
But once agents enter production, the core problems are:
reliable completion, evidence chains, governance, rollback, confirmation, and delivery semantics.

> **This is the space of Protocol Engineering.**

### The Claim

> I discovered the most underestimated architectural problem of the Agent era,
> and I am validating it with protocols, runtimes, and products.

## 6. Visual Redesign Principles

### Retain
- Dark Hero zone
- Registry annotations (sparse)
- Two-zone contrast model (dark identity / light reading)

### Change
- **Compress** Hero height
- **Reduce** registry annotation frequency (not every surface)
- **Simplify** cards from "complex system modules" to "evidence cards"
- **Brighten** reading zone — must be cleaner and more breathable
- **Shift** visual language from "showcasing complexity" to "proving clarity"

### Visual Keywords Upgrade

FROM: `protocol / registry / ledger / system density`
TO: `clarity / evidence / reliability / governed delivery`
