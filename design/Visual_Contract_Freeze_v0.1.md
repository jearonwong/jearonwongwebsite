# JearonWong.com Visual Contract Freeze v0.1

## Status
- Document Type: Execution lock / visual contract
- Authority Level: Subordinate to `Brand_Identity_Brief_v0.1` and `VI_System_Brief_v0.2_locked`
- Purpose: Freeze the homepage visual hierarchy, perceptual goals, visual tokens, forbidden patterns, and implementation red lines before UI execution

---

## 0. Canonical Inheritance
This document inherits the following locked identity constraints:
- **Brand Name:** Jearon Wong
- **Canonical Role Line:** Protocol Architect for the Agent Era
- **Canonical Thesis Line:** Defining how AI agents reliably deliver real work.
- **Canonical Proof Chain:**
  - MPLP
  - Cognitive OS
  - SoloCrew
  - Validation Lab

These must not be weakened, generalized, or replaced by generic founder language.

---

## 1. First-Impression Contract
The homepage must satisfy the following perceptual outcomes:

### 1.1 Three-Second Outcome
A new visitor must immediately register:
- this is a real person
- this person is high-signal
- this is not a generic blog or content archive

### 1.2 Five-Second Outcome
A new visitor must understand:
- what Jearon Wong is defining
- that the site is about reliable AI agent work delivery
- that the core frame is protocol / governance / AI-native delivery

### 1.3 Ten-Second Outcome
A new visitor must have seen:
- the proof chain
- at least the names MPLP / Cognitive OS / SoloCrew / Validation Lab
- enough evidence to conclude the site is grounded in real work, not abstract opinion alone

### 1.4 Archetype Rejection Requirement
The homepage must NOT be reasonably mistaken for:
- a legacy blog
- a literary essay site
- a generic founder homepage
- a generic AI SaaS landing page
- a consulting brochure page

---

## 2. Homepage Wireframe Freeze
The homepage must use this locked order:
1. Person
2. Thesis
3. Immediate Proof Signal
4. Signature Ideas
5. Why Now
6. Expanded Proof
7. Featured Writing
8. Playbooks Preview
9. Quiet Subscribe / Contact

### 2.1 Interpretation Rule
`Immediate Proof Signal` must appear in the first screenful or immediate post-hero zone.
It may take the form of: proof strip, proof pills, compact proof micro-cards, short evidence bar.
It must not require deep scrolling.

### 2.2 Section Weight Rule
The sections do not have equal importance.
Top hierarchy must be:
1. Person
2. Thesis
3. Immediate Proof
Everything below is secondary.

---

## 3. Hero Freeze

### 3.1 Hero Structure
The hero must be a true split hero.
**Left Side**
Must contain: eyebrow / micro-label, name, canonical role line or canonical thesis line depending on hierarchy choice, supporting sentence, 2 main CTAs maximum.
**Right Side**
Must contain: portrait anchor.
Portrait must be visually large enough to function as a memory device; portrait area must not read as a decorative side card.

### 3.2 Hero Copy Weight
The hero must feel: short, sharp, high-confidence, non-academic, non-salesy.
The hero must not start with: disclaimers, boundary language, abstract taxonomy, long explanatory paragraphs.

### 3.3 CTA Rule
Hero CTAs must be limited and hierarchical.
Recommended: Read the Thesis, Explore Projects.
Optional third CTA: Subscribe.
Do not place too many equal CTA choices in the first fold.

---

## 4. Portrait Contract

### 4.1 Portrait Centrality
Portrait is not optional decoration. It is part of the recognition mechanism.

### 4.2 Portrait Visual Style
Portrait must feel: editorial, premium, high-contrast, serious, founder / architect energy, current, not nostalgic.

### 4.3 Portrait Fallback
If no real portrait asset exists, use a locked fallback treatment:
- strong portrait frame
- monogram or minimal identity marker
- controlled signal/grid texture
- premium composition
- obvious future-replacement path

### 4.4 Forbidden Portrait Outcomes
- tiny portrait card
- soft empty placeholder block
- casual avatar
- stock-photo feeling
- decorative filler without identity function

---

## 5. Token Freeze

### 5.1 Color Behavior
The site must use a two-zone contrast model:
**Zone A — Hero / Identity Zone:** near-black or deep charcoal base, strong contrast, premium AI-era seriousness
**Zone B — Main Reading Zone:** cool off-white / light neutral, readable, calm, not beige-paper dominant
**Accent:** one restrained cool accent only, used for emphasis, links, focus, and signal elements, not used as a noisy branding field

### 5.2 Typography Behavior
**Primary Brand Typography:** modern sans / grotesk / neo-grotesk; sharp, current, systems-oriented
**Secondary Editorial Typography:** serif allowed only as accent; never as the dominant homepage voice
**Micro Layer:** mono optional; sparse, utility-like

### 5.3 Spacing Behavior
- first fold must breathe
- section transitions must be visibly different
- no compressed card-stack homepage
- whitespace must create rhythm, not emptiness

### 5.4 Surface Behavior
- cards may exist, but homepage must not feel card-led
- surfaces should feel premium and controlled
- avoid soft generic blog cards
- avoid SaaS-gloss cards

---

## 6. Navigation Freeze

### 6.1 Top Navigation Priority
Top nav must prioritize brand clarity over IA completeness.
Preferred visible top nav: Essays, Ideas, Playbooks, Projects, About

### 6.2 Demoted Items
The following may remain accessible but should not dominate top-nav clarity: Governance, Newsletter, Contact
These may appear in: footer, secondary nav, hero micro-links, mobile drawer

### 6.3 Label Rule
Visible label should prefer: Ideas.
If `/theories` remains as route, visible language may still use `Ideas`.

---

## 7. Section Rhythm Freeze

### 7.1 No Flat Homepage Rhythm
The homepage must not be a sequence of same-height, same-weight sections.

### 7.2 Rhythm Requirements
At minimum, the homepage must contain:
- one dominant first fold
- one high-contrast narrative band
- one proof-driven section
- one lighter editorial/content section
- one quiet closing section

### 7.3 Equal-Weight Card Limitation
Equal-weight cards are allowed only in lower information grids (signature ideas, featured essays, playbook previews, lower proof cards). They are forbidden as the dominant first-fold logic.

---

## 8. Forbidden Visual Patterns
The following are explicitly prohibited:
- beige-paper dominant homepage mood
- serif-led homepage identity
- old-blog aesthetics
- literary-essay mood as primary brand surface
- generic AI SaaS neon look
- card-stack homepage
- portrait as decorative filler
- too many equal-weight sections
- too many equal-weight CTAs
- first fold dominated by abstract explanation
- first fold dominated by warnings / disclaimers / non-goals
- generic founder branding that loses protocol-specific identity

---

## 9. Content-to-Visual Priority Rule
On the homepage, the order of communication priority is:
1. Who this person is
2. What he is defining
3. Why it matters now
4. What proves it
5. Where to go deeper

The order must NOT become: taxonomy -> theory dump -> archive -> proof later.

---

## 10. Implementation Tests
A design implementation passes only if all tests below are satisfied.

### 10.1 Person Test
At first glance, does the page feel centered on a specific person rather than a generic publication?
### 10.2 Thesis Test
Within 5 seconds, can a new visitor say what this person is trying to define?
### 10.3 Proof Test
Within 10 seconds, can a new visitor see the proof chain?
### 10.4 Archetype Rejection Test
Would a reasonable viewer mistake this page for an old blog, a literary journal, a startup SaaS template, or a generic founder profile? If yes, the implementation fails.
### 10.5 Visual Memory Test
After closing the page, can the viewer remember the person, the core thesis, and at least one proof artifact? If not, the implementation is too weak.

---

## 11. Handoff Rule for Codex / Designers
Any implementation wave must treat this freeze as non-negotiable for: first fold, portrait treatment, top hierarchy, proof timing, typography dominance, forbidden visual archetypes.
Small visual refinements are allowed. Structural contradiction is not allowed.

---

## 12. Change Control
Any change to the following requires a new brief revision: first fold structure, role line, thesis line, proof exposure timing, top-nav priority, portrait centrality, primary typography direction, two-zone contrast model.
